## 安卓开发中的常用组件

安卓开发中，UI 组件是构建用户界面的基本元素，通常定义在 XML 布局文件中，并通过 Java 或 Kotlin 代码进行逻辑控制。以下是一些常用组件的介绍，包括属性说明、简介和用法示例。

### 1. TextView 组件

#### 简介
`TextView` 是安卓中最基本的文本显示组件，用于在界面上显示文本内容。它可以设置文本样式、大小、颜色、对齐方式等，适用于显示静态文本信息。

#### 常用属性
| 属性名          | 必填 | 可选值                          | 示例            | 默认值       | 说明                     |
|-----------------|------|--------------------------------|-----------------|--------------|--------------------------|
| `layout_width`  | 是   | `wrap_content`, `match_parent`, 具体值（如 `100dp`） | `wrap_content`  | 无           | 组件宽度                 |
| `layout_height` | 是   | `wrap_content`, `match_parent`, 具体值（如 `50dp`）  | `wrap_content`  | 无           | 组件高度                 |
| `text`          | 否   | 任意字符串                     | `"Hello World"` | 空字符串     | 显示的文本内容           |
| `textColor`     | 否   | 颜色值（如 `#RRGGBB`）         | `#00ff00`       | 黑色         | 字体颜色                 |
| `textSize`      | 否   | 字体大小（如 `20sp`）          | `20sp`          | 系统默认     | 字体大小，推荐用 `sp` 单位 |
| `textStyle`     | 否   | `normal`, `bold`, `italic`     | `bold`          | `normal`     | 文本样式（普通、粗体、斜体） |
| `gravity`       | 否   | `center`, `left`, `right`, `top`, `bottom` 等 | `center`        | `left|top`   | 文本对齐方式             |
| `maxLines`      | 否   | 整数值                         | `2`             | 无限制       | 最大显示行数             |
| `ellipsize`     | 否   | `start`, `middle`, `end`, `marquee` | `end`           | 无           | 文本过长时的省略方式     |

#### 示例用法
在 XML 布局文件中定义 `TextView`：
```xml {.line-numbers highlight=1-3}
<TextView
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Hello, Android!"
    android:textColor="#FF0000"
    android:textSize="20sp"
    android:textStyle="bold"
    android:gravity="center"
    android:maxLines="1"
    android:ellipsize="end" />
```

在 Java 代码中动态设置 `TextView`：
```java
TextView textView = findViewById(R.id.textView);
textView.setText("Dynamic Text");
textView.setTextColor(Color.parseColor("#00FF00"));
textView.setTextSize(18);
```

### 2. EditText 组件

#### 简介
`EditText` 是用于接收用户输入的文本输入框，继承自 `TextView`，支持单行或多行输入，常用于表单、搜索框等场景。

#### 常用属性
| 属性名          | 必填 | 可选值                          | 示例            | 默认值       | 说明                     |
|-----------------|------|--------------------------------|-----------------|--------------|--------------------------|
| `layout_width`  | 是   | `wrap_content`, `match_parent`, 具体值 | `match_parent`  | 无           | 组件宽度                 |
| `layout_height` | 是   | `wrap_content`, `match_parent`, 具体值 | `wrap_content`  | 无           | 组件高度                 |
| `hint`          | 否   | 任意字符串                     | `"Enter name"`  | 空字符串     | 输入框提示文本           |
| `inputType`     | 否   | `text`, `number`, `password`, `email`, `phone` 等 | `password`      | `text`       | 输入类型，决定键盘样式   |
| `maxLines`      | 否   | 整数值                         | `1`             | 无限制       | 最大显示行数，1 为单行   |
| `imeOptions`    | 否   | `actionDone`, `actionSearch`, `actionNext` 等 | `actionDone`    | 无           | 软键盘的操作按钮类型     |

#### 示例用法
在 XML 布局文件中定义 `EditText`：
```xml
<EditText
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:hint="Enter your password"
    android:inputType="textPassword"
    android:maxLines="1"
    android:imeOptions="actionDone" />
```

在 Java 代码中获取用户输入：
```java
EditText editText = findViewById(R.id.editText);
String userInput = editText.getText().toString();
editText.setOnEditorActionListener((v, actionId, event) -> {
    if (actionId == EditorInfo.IME_ACTION_DONE) {
        Toast.makeText(this, "Input: " + userInput, Toast.LENGTH_SHORT).show();
        return true;
    }
    return false;
});
```

### 3. Button 组件

#### 简介
`Button` 是安卓中用于触发操作的按钮组件，用户点击后可以执行特定逻辑，常用于提交表单、导航等。

#### 常用属性
| 属性名          | 必填 | 可选值                          | 示例            | 默认值       | 说明                     |
|-----------------|------|--------------------------------|-----------------|--------------|--------------------------|
| `layout_width`  | 是   | `wrap_content`, `match_parent`, 具体值 | `wrap_content`  | 无           | 组件宽度                 |
| `layout_height` | 是   | `wrap_content`, `match_parent`, 具体值 | `wrap_content`  | 无           | 组件高度                 |
| `text`          | 否   | 任意字符串                     | `"Click Me"`    | 空字符串     | 按钮显示的文本           |
| `background`    | 否   | 颜色值、drawable 资源          | `#FF0000`       | 系统默认     | 按钮背景样式             |
| `onClick`       | 否   | 方法名                         | `onButtonClick` | 无           | 点击事件处理方法名       |

#### 示例用法
在 XML 布局文件中定义 `Button`：
```xml
<Button
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Click Me"
    android:background="#00FF00"
    android:onClick="onButtonClick" />
```

在 Java 代码中设置点击事件：
```java
Button button = findViewById(R.id.button);
button.setOnClickListener(v -> {
    Toast.makeText(this, "Button Clicked!", Toast.LENGTH_SHORT).show();
});

// 或者通过 XML 的 onClick 属性绑定方法
public void onButtonClick(View view) {
    Toast.makeText(this, "Button Clicked via XML!", Toast.LENGTH_SHORT).show();
}
```

### 4. RadioButton 组件

#### 简介
`RadioButton` 是单选按钮组件，通常与 `RadioGroup` 配合使用，用于让用户从多个选项中选择一个。

#### 常用属性
| 属性名          | 必填 | 可选值                          | 示例            | 默认值       | 说明                     |
|-----------------|------|--------------------------------|-----------------|--------------|--------------------------|
| `layout_width`  | 是   | `wrap_content`, `match_parent`, 具体值 | `wrap_content`  | 无           | 组件宽度                 |
| `layout_height` | 是   | `wrap_content`, `match_parent`, 具体值 | `wrap_content`  | 无           | 组件高度                 |
| `text`          | 否   | 任意字符串                     | `"Option 1"`    | 空字符串     | 单选按钮旁边的文本       |
| `checked`       | 否   | `true`, `false`                | `true`          | `false`      | 是否默认选中             |

#### 示例用法
在 XML 布局文件中定义 `RadioButton` 和 `RadioGroup`：
```xml
<RadioGroup
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:orientation="vertical">

    <RadioButton
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Option 1"
        android:checked="true" />

    <RadioButton
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Option 2" />
</RadioGroup>
```

在 Java 代码中监听选择变化：
```java
RadioGroup radioGroup = findViewById(R.id.radioGroup);
radioGroup.setOnCheckedChangeListener((group, checkedId) -> {
    RadioButton selectedRadio = findViewById(checkedId);
    Toast.makeText(this, "Selected: " + selectedRadio.getText(), Toast.LENGTH_SHORT).show();
});
```

### 5. ImageView 组件

#### 简介
`ImageView` 用于在界面上显示图片，支持从资源文件、URL 或本地文件加载图片，常用于图标、背景图等场景。

#### 常用属性
| 属性名          | 必填 | 可选值                          | 示例            | 默认值       | 说明                     |
|-----------------|------|--------------------------------|-----------------|--------------|--------------------------|
| `layout_width`  | 是   | `wrap_content`, `match_parent`, 具体值 | `100dp`         | 无           | 组件宽度                 |
| `layout_height` | 是   | `wrap_content`, `match_parent`, 具体值 | `100dp`         | 无           | 组件高度                 |
| `src`           | 否   | 图片资源（如 `@drawable/image`）| `@drawable/icon`| 无           | 图片资源                 |
| `scaleType`     | 否   | `fitCenter`, `centerCrop`, `fitXY` 等 | `centerCrop`    | `fitCenter`  | 图片缩放和裁剪方式       |

#### 示例用法
在 XML 布局文件中定义 `ImageView`：
```xml
<ImageView
    android:layout_width="100dp"
    android:layout_height="100dp"
    android:src="@drawable/my_image"
    android:scaleType="centerCrop" />
```

在 Java 代码中动态设置图片：
```java
ImageView imageView = findViewById(R.id.imageView);
imageView.setImageResource(R.drawable.my_image);
// 或者从网络加载图片（需使用库如 Glide 或 Picasso）
Glide.with(this).load("https://example.com/image.jpg").into(imageView);
```

### 6. AlertDialog 组件

#### 简介
`AlertDialog` 是一种弹出式对话框组件，用于向用户显示重要的提示信息、确认操作或提供简单的选择。它通常包含标题、消息内容以及一个或多个按钮（如“确认”和“取消”），广泛用于用户交互场景。

#### 常用属性与方法
由于 `AlertDialog` 通常是通过代码动态创建的，而不是在 XML 布局文件中定义，因此其属性更多是通过 `Builder` 类或方法设置的。以下是常用属性和方法：

| 属性/方法名          | 必填 | 示例值/用法                                  | 默认值       | 说明                              |
|----------------------|------|----------------------------------------------|--------------|-----------------------------------|
| `setTitle()`         | 否   | `"Warning"`                                 | 无           | 设置对话框标题                   |
| `setMessage()`       | 否   | `"Are you sure you want to delete this?"`   | 无           | 设置对话框的消息内容             |
| `setPositiveButton()`| 否   | `"OK", (dialog, which) -> { /* action */ }` | 无           | 设置确认按钮及其点击监听器       |
| `setNegativeButton()`| 否   | `"Cancel", (dialog, which) -> { /* action */ }` | 无        | 设置取消按钮及其点击监听器       |
| `setNeutralButton()` | 否   | `"Later", (dialog, which) -> { /* action */ }` | 无         | 设置中立按钮及其点击监听器       |
| `setCancelable()`    | 否   | `true` 或 `false`                           | `true`       | 设置对话框是否可通过返回键取消   |
| `setIcon()`          | 否   | `R.drawable.icon`                           | 无           | 设置对话框图标                   |
| `setView()`          | 否   | `customView`                                | 无           | 设置自定义布局视图               |

#### 示例用法
```java
    AlertDialog.Builder dlg = new AlertDialog.Builder(Activity3.this);
    dlg.setTitle("请确定您的操作");
    dlg.setMessage("真的要这么做吗？");
    dlg.setCancelable(true);
    // 肯定操作按钮
    dlg.setPositiveButton("确定", new DialogInterface.OnClickListener() {
        @Override
        public void onClick(DialogInterface dialog, int which) {
            Toast.makeText(Activity3.this, "操作确认", Toast.LENGTH_SHORT).show();
        }
    });
    dlg.show(); // 显示
```

#### 注意事项
1. **上下文（Context）**：创建 `AlertDialog` 时需要传入当前活动的上下文（如 `this` 或 `getApplicationContext()`）。
2. **按钮数量**：`AlertDialog` 最多支持三个按钮（Positive、Negative 和 Neutral），过多按钮建议使用自定义布局。
3. **显示和关闭**：使用 `show()` 方法显示对话框，使用 `dismiss()` 方法关闭对话框。
4. **线程安全**：`AlertDialog` 必须在 UI 线程中操作，若在子线程中需通过 `Handler` 或 `runOnUiThread()` 更新。
