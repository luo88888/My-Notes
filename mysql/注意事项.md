### MySQL 注意事项

#### 1. 大小写

##### 1.1 账户名是否区分大小写

在 MySQL 中，**账户名**是否区分大小写取决于操作系统的文件系统和 MySQL 的配置。

- **Linux/Unix 系统**：默认区分大小写。
    - 例如，`User1` 和 `user1` 是两个不同的账户。
- **Windows 系统**：默认不区分大小写。
    - 例如，`User1` 和 `user1` 被视为同一个账户。

##### 1.2 数据库名和表名是否区分大小写

在 MySQL 中，**数据库名**和**表名**是否区分大小写取决于操作系统及文件系统对大小写的敏感性以及MySQL的配置。

- **Linux/Unix系统**：默认情况下，MySQL 对数据库名和表名是区分大小写的，因为这些系统的文件系统（如ext4）通常是区分大小写的。这意味着 `myDatabase` 和 `MyDatabase` 会被视为两个不同的数据库。
- **Windows 系统**：由于其文件系统（NTFS或FAT32）不区分大小写，MySQL 在这类系统上也不区分数据库名和表名的大小写。所以在 Windows 上，`myTable` 和 `MYTABLE` 被视为同一个表。
- **macOS系统**：从 MySQL 8.0 开始，默认情况下表名的比较是大小写敏感的，这与旧版本的行为有所不同。但这个行为也依赖于操作系统的设置和 MySQL 的配置。

- 此外，MySQL 有一个参数叫做 `lower_case_table_names`，它可以影响**数据库名**和**表名**（不影响**账户名**）的大小写处理：

    - 设置为0（默认在 Linux/Unix），表明数据库名和表名按指定的大小写存储，并且区分大小写。
    - 设置为1（默认在 Windows），表明数据库名和表名以小写形式存储，不论输入时使用的是大写还是小写。
    - 设置为2（默认在 macOS），表明数据库名和表名按指定的大小写存储，但在比较时不区分大小写。

因此，在设计应用或数据库时，了解部署环境的具体情况并考虑相应的大小写策略是很重要的，以便避免潜在的问题，最好采用一致的小写命名规则，并使用下划线来分隔单词。
